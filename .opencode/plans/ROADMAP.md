# VeriSure 产品路线图

## 一、产品定位

### 1.1 核心价值主张

**"让等保合规从'纸上谈兵'变成'实战验证'"**

- **传统等保**：配置打勾 = 合规，实际防线无效
- **VeriSure**：白盒配置 + 黑盒渗透交叉验证，打出"纸上合规"原形

### 1.2 目标用户

| 角色 | 核心诉求 | 产品价值 |
|------|----------|----------|
| 企业老板/CIO | 合规不出事、成本低 | 快速过等保、避免罚款和停业风险 |
| IT/安全负责人 | 省事、不出锅 | 自动化检查、整改方案直接给 |
| 安全合规经理 | 管理多项目、协调整改 | 统一看板、工单跟踪、持续监控 |
| 等保测评机构 | 提高测评效率、降低返工 | 预检报告减少现场工作量 |

### 1.3 产品愿景

| 阶段 | 定位 | 关键能力 |
|------|------|----------|
| 当前 | 等保自检工具 | AI Agent + MCP 自动化检查 |
| 1-2 年 | 合规操作系统 | 持续监控 + 整改闭环 + 多标准支持 |
| 3-5 年 | 合规生态平台 | 接入测评机构、云厂商、企业客户 |

### 2.2 未完成功能

#### 🔴 紧急（P0）
- [ ] **hydra 弱口令检测未真正实现**
  - 当前：硬编码返回空结果
  - 影响：等保 8.1.4.1 身份鉴别无法检测
  
- [ ] **缺少白盒配置核查**
  - 当前：只有黑盒扫描
  - 影响：无法检查密码策略、审计配置、权限分配等
  - 等保覆盖率：约 15%（仅技术检测的部分项）

#### 🟠 高优先级（P1）
- [ ] **访谈管理缺失**
  - 等保要求"访谈+核查+测试"三维验证
  - 当前只有"测试"，缺少"访谈"和"核查"
  
- [ ] **报告生成不完善**
  - 当前：只有简单的扫描结果
  - 缺少：等保测评报告模板、不符合项清单、整改建议

- [ ] **OCR 工具未集成**
  - ocr-server 已部署，但未注册为能力
  - AI 无法调用截图分析功能

#### 🟡 中优先级（P2）
- [ ] **整改跟踪缺失**
  - 无法管理不符合项的整改和复测
  - 缺少整改建议生成

- [ ] **缺少更多检测工具**
  - Web 应用扫描（OWASP ZAP）
  - 数据库安全检测
  - 网络设备检测（SNMP）
  - Windows 安全检测

#### 🟢 低优先级（P3）
- [ ] **流程管理缺失**
  - 没有六阶段流程管理（定级→备案→差距分析→测评→整改→复测）
  - 没有证据管理

- [ ] **管理项辅助缺失**
  - 等保 273 项要求中，约 130 项是管理要求
  - 缺少制度文档模板、人员权限矩阵等

```python
async def ssh_config_check(params: Dict[str, Any]) -> Dict[str, Any]:
    # 通过 SSH 登录到目标执行配置检查
    checks = [
        {"name": "password_policy", "cmd": "cat /etc/login.defs | grep PASS_MAX_DAYS"},
        {"name": "ssh_config", "cmd": "cat /etc/ssh/sshd_config | grep PermitRootLogin"},
        {"name": "audit_config", "cmd": "systemctl status auditd"},
        # ...
    ]
    
    results = {}
    for check in checks:
        # 执行远程命令并收集结果
        results[check["name"]] = await execute_ssh_command(...)
    
    return {"checks": results}
```

## 五、商业模式

### 5.1 版本矩阵

| 维度 | 轻量合规版 | 高级合规+验证版 |
|------|------------|-----------------|
| 核心诉求 | 快速自检，成本低 | 实战合规，防真实攻击 |
| 接入方式 | 仅提供公网 IP / 域名 | 授权云 AK/SK 或安装 Agent |
| 可用工具池 | 黑盒探测工具集 | 轻量版全量 + 白盒读取 + 渗透验证 |
| 核心卖点 | "5分钟出外网风险" | "配置合规不算合规，防线防住才是合规" |

### 5.2 定价策略

| 维度 | 轻量合规版 | 高级合规+验证版 |
|------|------------|-----------------|
| 计费模式 | 按年订阅 | 按资产数 x 年 |
| 参考价格 | 二级：9,800/年；三级：19,800/年 | 二级：98,000/年；三级：198,000/年 |
| 免费钩子 | 首次免费扫描 1 个 IP | 14 天全功能试用 |

### 5.3 渠道策略

| 渠道 | 合作模式 | 目标 |
|------|----------|------|
| 等保测评机构 | 作为"预检工具" | 降低测评不通过率 |
| 云厂商市场 | 上架阿里云/华为云/腾讯云市场 | 获取精准流量 |
| 安全集成商 | OEM/白标 | 覆盖中长尾客户 |

## 六、法律合规框架

### 6.1 三级授权体系

| 授权级别 | 风险等级 | 适用操作 | 用户流程 |
|----------|----------|----------|----------|
| L1 资产所有权验证 | 低风险 | 端口扫描、配置核查 | DNS TXT / 文件验证 |
| L2 在线确认 | 中风险 | 弱口令检测 | 勾选同意 + 短信验证码 |
| L3 正式授权 | 高风险 | 渗透测试 | 电子授权书 + 电子签章 |

### 6.2 操作审计

| 审计项 | 存储方式 | 保留期限 |
|--------|----------|----------|
| MCP 工具调用日志 | WORM 存储 | >= 6 个月 |
| 渗透测试全程流量 | PCAP 文件 | >= 3 个月 |
| 授权书签署记录 | 区块链存证 | 永久 |
| 用户操作行为日志 | 不可篡改日志 | >= 6 个月 |

## 八、风险与应对

### 8.1 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| LLM 服务不可用 | 无法生成结果 | 降级为纯规则引擎 |
| MCP 工具超时 | 扫描失败 | 重试 + 降级 |
| 容器网络限制 | 无法准确判断主机状态 | 改进判断逻辑 |

### 8.2 商业风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 用户对等保不感兴趣 | 无法获客 | 强调"实战验证"价值 |
| 测评机构不认可 | 无法进入渠道 | 提供预检报告，降低测评工作量 |
| 法律合规风险 | 被追责 | 三级授权体系 + 操作审计 |
